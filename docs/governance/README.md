# Project governance

These documents define how architecture and milestone decisions become accountable.
They deliberately distinguish a documented proposal, an implementation review, and a
human sign-off. A checked task, passing CI, or an automated review is evidence, but is
not by itself approval.

- [Ownership model](ownership.md) — accountable roles, current assignments, and
  independence requirements.
- [Milestone review process](milestone-reviews.md) — entry criteria, evidence,
  decisions, and sign-off records.
- [Milestone review template](milestone-review-template.md) — the required record for
  each milestone gate.

The project currently has a single maintainer and no named role assignments recorded
in the repository. Until the ownership register records a consenting person, its roles
remain unassigned and the current maintainer is only the acting owner. Acting ownership
allows work to continue but cannot satisfy gates that require independent or external
review.
